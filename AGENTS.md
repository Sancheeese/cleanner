# Agent Instructions

This is a personal Android app project owned by Sancheeese.

## Git Workflow

- Work directly on the `main` branch.
- Do not create feature branches unless the user explicitly asks for one.
- After completing a requested change, commit it to `main`.
- Push committed changes to `origin/main` after committing.
- Keep commits focused and use clear commit messages.

## Collaboration Notes

- Treat this repository as a solo-development project.
- Prefer simple, direct workflows over pull-request-style branching.
- Before committing, check `git status` and avoid including unrelated user changes.
